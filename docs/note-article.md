## あなたのVault、こんな状態になっていませんか？

Obsidianを長く使っている人ほど、以下の症状に心当たりがあるはずです。

### 症状1: ノートの孤立化

「書いたけど、どこにもリンクされていないノート」が大量にある状態。

Obsidianの真価は双方向リンクにあります。でも、ノートを書くときに毎回「このノートはどこにリンクすべきか？」を考えるのは面倒です。結果として、**書きっぱなしのノートが増えていく**。

Obsidian Optimizerは、Vault全体のリンク構造を解析し、**どのノートからもリンクされず、自分からも他のノートにリンクしていない「完全孤立ノート」を自動検出**します。

検出した孤立ノートは、ワンクリックで適切なMOCに接続できます。キーワードベースの自動分類ルールが、ノートの内容に合ったMOCを提案してくれます。さらにv4.3では**AIがノートの内容を読んで最適なリンク先を提案**してくれます。

### 症状2: リンク切れの放置

ノートをリネームしたり移動したりすると、リンクが切れます。Obsidianは[[リンク先]]をファイル名で解決するため、ファイル名が変わるとリンクが壊れるのです。

数個のリンク切れなら手動で直せますが、数十個、数百個となると現実的ではありません。

Obsidian Optimizerは壊れたリンクを検出するだけでなく、**Levenshtein距離（編集距離）と部分一致アルゴリズムを使って、正しいリンク先の候補を自動提案**します。「このリンク、たぶんこっちのファイルじゃないですか？」と教えてくれるわけです。

### 症状3: ゴミファイルの蓄積

Obsidianでは、新規ノートを開いて何も書かずに閉じると空のファイルが残ります。「とりあえず作ったけど書かなかったノート」「メモを別のノートに移動して空になったノート」——こうした**実質的にゴミのファイルがVaultに堆積していく**のです。

Obsidian Optimizerのゴミファイル検出は、単にファイルサイズだけでなく、**フロントマターとコメントを除去した後の実質的なテキスト量**で判定します。フロントマターだけ残っているような「見かけ上は中身があるけど実質空」のファイルも見逃しません。

### 症状4: MOCが作れない / 作っても更新できない

MOC（Map of Content）は、ノートを整理する最も強力な仕組みです。テーマごとの「目次ノート」を作り、関連ノートへのリンクを集約する。これがあるだけで、Vaultの見通しは劇的に改善します。

でも、**MOCの作成と維持は面倒**です。どのノートをどのMOCに入れるか、手動で判断して、リンクを書いて、定期的に新しいノートを追加して——。ノートが1,000個を超えると、この作業だけで何時間もかかります。

Obsidian Optimizerは、**5種類のテンプレート + 3つの作成モード + AI自動分類で、MOCを数秒で生成**します。しかも、新しいノートを追加したらMOCも自動で更新できます。

### 症状5: Vaultの全体像が見えない

ノートが増えるほど、「自分のVaultが今どういう状態なのか」が分からなくなります。

- 全部で何個のノートがあるのか
- どれくらいが孤立しているのか
- どのフォルダが肥大化しているのか
- 最近の更新傾向はどうなっているのか
- 各ノートの「価値」はどれくらいなのか

こうした**メタ情報**を把握できていないと、整理の方針も立てられません。

Obsidian Optimizerのダッシュボードは、Vaultの「健康診断書」です。健全度スコア（0〜100）、統計カード、更新ヒートマップ、フォルダ分布チャート、推移グラフ、ノートスコア、実績バッジで、Vaultの全体像をひと目で把握できます。さらに**AIが週次インサイトを自動生成**し、「今週はこのトピックが多かった」「このノートを深掘りすべき」といったアドバイスもくれます。

## 2. ワンクリックスキャン — 6種類の問題を同時検出

ダッシュボードの「スキャン」ボタンを押すと、Vault全体を解析して以下の6カテゴリの問題を検出します。

### ⚠️ 孤立ノート

他のノートからリンクされず、自身も他のノートへリンクしていないノート。検出後、ワンクリックでMOCに接続するか、不要であれば削除できます。一括操作にも対応しており、チェックボックスで複数選択して一気にMOC接続することも可能です。

各ノートにはプレビューボタン（👁️）がついており、**統一プレビューモーダル**で中身を確認してから判断できます。プレビューモーダルでは、ノートの内容表示に加え、AI要約・タグ提案・リンク提案・品質レビューなどのAI機能にもアクセスできます。

### 📅 放置ノート

一定期間（デフォルト180日）更新されていないノート。「書いたまま忘れている」ノートを掘り起こします。一括アーカイブ機能で、`99 Archive`フォルダに移動できます。一括プレビュー機能で、選択した放置ノートの中身をまとめて確認することも可能です。放置日数は設定画面でカスタマイズ可能です。

### 🗑️ ゴミファイル

実質的に中身のないファイル。判定基準は3つ:
- ファイルサイズが最小バイト数（デフォルト5バイト）未満
- フロントマター・コメント除去後のテキストが最小文字数（デフォルト20文字）未満
- ファイル名に無題キーワード（`untitled`、`無題`など）を含む

処理方法は「完全削除」と「Archiveフォルダに移動」から選択できます。

### 💔 壊れたリンク

ターゲットファイルが存在しない内部リンク。リンク元ファイル、壊れたリンク先、修復候補（上位3件）を表示。Levenshtein距離と部分一致で候補を算出するため、「ファイル名をちょっと変えただけ」のケースはほぼ正確にマッチします。

「候補ありを一括修復」ボタンで、修復候補が存在するリンクをすべて一括修復することも可能です。

### 📋 重複ノート候補

ファイル名の類似度、ファイルサイズ、コンテンツハッシュから重複の可能性があるノートをペアで表示。マージプレビューを確認してから統合できます。

さらに**AI重複検出**機能では、AIが意味的に重複するノートを検出します。ファイル名が異なっていても、内容が似ているノートを見つけ出します。

### 🖼️ 孤立画像・添付ファイル

どのノートからもリンクされていない画像（PNG, JPG, GIF）、PDF、動画ファイルを検出。Obsidianにスクリーンショットをペーストすると画像ファイルが自動作成されますが、ノートを削除しても画像は残ります。こうした**参照されない添付ファイルがVaultの容量を食っている**ケースは非常に多いです。

合計ファイルサイズも表示されるので、「孤立画像だけで500MB使っていた」といった気づきが得られます。

## 6. MOC自動生成 — ノート整理の「本丸」

MOC（Map of Content）は、Obsidianにおけるノート整理の核心です。

テーマごとの「目次ノート」を作り、関連ノートへのリンクを集約する。これがあるだけで、「あのノートどこだっけ？」はなくなり、Vault全体に構造が生まれます。

でも、MOCを手動で作るのは大変です。

- どのノートをどのMOCに入れるか、ひとつずつ判断する必要がある
- 新しいノートを書いたら、既存のMOCに手動で追加しなければならない
- そもそも「どんなMOCを作るべきか」の設計から考えなければならない

Obsidian Optimizerは、この作業を**3つのモード**で自動化します。

### 6-1. テンプレート — 5種類から選ぶだけ

あらかじめ用意された5種類のテンプレートから選ぶだけで、即座にMOCが生成されます。

| テンプレート | 特徴 | こんな人向け |
|---|---|---|
| **標準MOC** | Pinned Notes + Dataviewクエリ付きの本格的なMOC | 万能。迷ったらこれ |
| **シンプルMOC** | フロントマター + リンクリストのみのシンプル構成 | 軽量に始めたい人 |
| **フォルダMOC** | フォルダ内ファイルを自動集約するMOC | フォルダ整理派 |
| **Zettelkasten MOC** | 双方向リンク中心、Backlinks重視の構成 | Zettelkastenユーザー |
| **タグベースMOC** | 特定タグを持つノートを自動収集・統計表示 | タグ活用派 |

テンプレートの中では、以下の変数が自動的に埋め込まれます:

- `{{NAME}}` — MOCのタイトル
- `{{TAGS}}` — フロントマターのタグ
- `{{AUTO_TAGS}}` — 自動推定されたタグ
- `{{AUTO_FOLDERS}}` — 関連フォルダ一覧
- `{{RELATED_MOCS}}` — リンク先の既存MOC
- `{{DESCRIPTION}}` — MOCの説明文（空欄時はトピック名から自動生成）

### 6-2. ウィザードモード — ステップバイステップで迷わない

MOCを初めて作る人や、「何から始めればいいか分からない」人のためのモード。5つのステップで順番に案内します。

**ステップ1**: テンプレートを選ぶ
**ステップ2**: MOCの名前と説明を入力
**ステップ3**: タグとフォルダを設定
**ステップ4**: AIスマート提案で関連ノートを確認（🤖 AI自動構成ボタンでAIにMOC構成を任せることも可能）
**ステップ5**: プレビューで仕上がりを確認して作成

「前へ」「次へ」ボタンで自由に行き来でき、どのステップからでもやり直せます。

### 6-3. 通常モード — 全項目を自由に設定

上級者向けのモード。テンプレート選択、基本設定、詳細設定（自動集約フォルダ・タグ・関連MOC・CSSクラス）、スマートノート候補、プレビューをすべてひとつの画面で自由に操作できます。

**ワンクリック プリセット**も搭載。「フォルダ別MOC」「タグ別MOC」「プロジェクトMOC」のプリセットボタンを押すだけで、よく使うパターンのフォーム入力が自動で完了します。

### 6-4. 一括生成モード — 全フォルダに一気にMOC作成

チェックを入れたフォルダごとにMOCファイルを自動作成。「Vaultの全フォルダにMOCを一気に作りたい」というケースに対応。保存先フォルダの指定も可能です。

### 6-5. スマート提案 — AIがノートを自動分類

MOCの名前、タグ、フォルダを入力すると、**Vault内の全ノートを解析して、そのMOCに関連しそうなノートを自動で提案**します。

アルゴリズムの仕組み:
1. **タグマッチング**: 入力タグを持つノートを抽出
2. **フォルダマッチング**: 対象フォルダ内のノートを抽出
3. **キーワードマッチング**: MOC名に含まれるキーワードで、Vault全体をファジー検索
4. **スコアリング**: 上記3つの結果を統合し、関連度スコアで順位づけ

さらに**🤖 AI自動構成**ボタンを押すと、AIがノートの内容を読んで、最適なMOC構成を自動で提案してくれます。

### 6-6. キーワードベース自動分類ルール

Obsidian Optimizerには、ノートを自動でカテゴリ分けするためのルールが組み込まれています。

| カテゴリ | キーワード例 | 生成MOC名 |
|---|---|---|
| AI・LLM | ai, gpt, claude, llm, dify, langchain | MOC - AI Tools & Platforms |
| Threads | threads, スレ | MOC - Threads |
| マーケティング | マーケティング, seo, cvr, コンバージョン | MOC - マーケティング |
| 開発・技術 | python, react, javascript, api | MOC - Dev Skills |
| SNS | twitter, instagram, tiktok | MOC - SNS Strategy |
| ビジネス | ビジネス, 起業, saas | MOC - Business Strategy |
| 学習 | 学習, 勉強, 読書 | MOC - Learning |

これらのルールは設定画面からカスタマイズ可能です。

### 6-7. 既存MOCの更新 + タグベースMOC候補

**既存MOC更新機能**は、すでに作成されたMOCファイルを再分析し、新しく追加すべきノートを自動で提案します。MOCのフロントマターに設定された auto_folders / auto_tags を元に検出。

**タグベースMOC候補**は、よく使われているタグを分析し、MOC化すべき候補を自動提案します。

**MOC関連性マップ**は、MOC同士のリンク関係を可視化。どのMOCとどのMOCがつながっているかを俯瞰できます。

### 6-8. テンプレート管理 — JSON書き出し/読み込み

よく使うMOC設定は、カスタムテンプレートとして保存できます。さらに**JSON形式でエクスポート/インポート**に対応しているので、他のユーザーとテンプレートを共有できます。「この人のMOC構成が良さそうだからインポートしよう」といった使い方が可能です。

## 9. 整理ツール（7種） — Vaultの「大掃除」を自動化

整理ツールタブには、ノートの整理・整頓を支援する**7つの専門ツール**が搭載されています。「🔍 全ツールを一括スキャン」ボタンで、7つすべてを一度にまとめてチェックすることも可能です。

### ツール1: 📝 メタデータ補完（おすすめ）

ノートの先頭にある管理情報（フロントマター）が不足していないかチェックし、自動で補完します。作成日、タグ、タイプなどの不足フィールドを検出し、ワンクリックで追加。

### ツール2: 📂 Inbox整理（おすすめ）

「00 Inbox」フォルダに溜まっているノートを、内容やタグに基づいて適切なフォルダに自動振り分け。「とりあえずInboxに入れておく」派の人にとって、週末の整理作業が一瞬で終わります。

### ツール3: ✏️ タイトル修正

ファイル名とノート内の見出し（# タイトル）が一致しないノートを検出。ファイル名を正しいタイトルに変更し、他のノートからのリンクも自動で更新します。

さらに**🏷️ AIタイトル提案**機能では、「untitled」「無題」「メモ」など内容を反映していないタイトルのノートをAIが検出し、適切なタイトルを提案してくれます。

### ツール4: 🔗 リンク修正

ノート間リンクの大文字・小文字の不一致（例: [[Note]] と [[note]]）を検出し、一括で統一します。

### ツール5: 🗑️ 空フォルダ削除

ノートが1つも入っていないフォルダを検出して一括削除。ノートの移動・削除後に空になったフォルダが溜まるのを防ぎます。

### ツール6: 📋 TODO抽出

ノート内の未完了TODO（`- [ ]`）を全Vault横断で抽出し、一覧表示。タスクタブとの連携も可能です。

### ツール7: ✂️ 長文ノート分割提案

指定文字数を超えるノートを見出し（H2/H3）単位で自動分割する提案を表示。1つの巨大なノートを適切なサイズに分割できます。

### アーカイブ・統合・構造テンプレート

整理ツールタブには4つのサブタブがあり、上記の7ツールに加えて:

- **📦 アーカイブ** — 更新頻度・リンク数からスコア算出し、アーカイブ候補をランキング表示
- **🔀 統合** — 重複ノートのマージ（統合先の末尾に内容追加、被リンク自動リダイレクト）
- **🏗️ テンプレート** — PARA / Zettelkasten / Johnny.Decimal の構造テンプレート適用

## 12. ナレッジグラフ分析 — 隠れた関係を発見する

ノートを整理するだけでなく、**ノート間の「見えない関係性」を発見する**のが、ナレッジグラフ分析です。

## 14. 構造テンプレート — Vaultをメソッドに準拠させる

「ノートの整理方法は分かったけど、そもそもフォルダ構造をどうすればいいのか分からない」

そんな人のために、世界的に実績のある3つの知識管理メソッドのフォルダ構造を、ワンクリックで適用できます。

### PARAメソッド

Tiago Forteが提唱した知識管理の世界標準。

| フォルダ | 役割 |
|---|---|
| `01 Projects` | 現在進行中のプロジェクト |
| `02 Areas` | 継続的な責任分野（健康、財務、キャリアなど） |
| `03 Resources` | 参考資料、学習ノート |
| `04 Archives` | 完了したもの、不要になったもの |

### Zettelkasten

Niklas Luhmannが実践した「知識カード」方式。フォルダ階層を最小限にし、双方向リンクとタグで構造化するアプローチ。フリート・文献・恒久ノートの成熟度別に分類します。

### Johnny.Decimal

10-19、20-29、30-39...のような10進数命名規則で厳密にカテゴリ分けする方式。大規模Vault向け。

### 適用フロー

1. **「Vault構造を分析」**ボタンで現在のフォルダ構造を検出
2. **推奨テンプレートを提案**（現在の構造に最も近いものを自動判定）
3. **プレビュー**で作成予定フォルダと既存ファイルのマッピングを確認
4. **「適用」**で不足フォルダを作成 + 既存ノートをキーワードマッチングで自動配置

**ノートの削除は一切行いません。**フォルダの作成とノートの移動のみです。適用前に必ずプレビューで影響範囲を確認してください。

## 15. ノート統合 — 安全なマージ

「同じトピックについて2つのノートに分かれて書いてしまった」——よくあることです。

ノート統合機能は、2つのノートを選択して安全にマージします。

1. 2つのノートをドロップダウンから選択
2. マージプレビューを確認（ノートA + セパレータ + ノートB、統合元の注釈付き）
3. 確認後にマージ実行
4. ノートBへのリンクは自動的に新しいマージファイルにリダイレクト
5. バックアップが自動作成

**リンクの自動リダイレクト**がポイントです。ノートBを削除するだけだと、ノートBへのリンクが全て切れてしまいます。Obsidian Optimizerは、リンク元のファイルを自動で更新し、マージ後のファイルを指すように書き換えます。

## 16. スマートアーカイブ提案

「このノート、もう使わないかも？」をスコアベースで提案します。

アルゴリズムは2つの要素を見ています:
- **更新頻度**: 最終更新からの経過日数
- **リンク状態**: 他のノートからの参照の有無

両方の加重平均でスコアを算出し、アーカイブ候補をランキング表示。個別プレビューで中身を確認してから、ワンクリックでアーカイブ（`99 Archive`フォルダに移動）できます。

## 17. 安全設計 — 「消してしまった」を防ぐ

Vaultのファイルを直接操作するアプリだからこそ、安全設計は最優先事項です。

### 自動バックアップ

設定でONにしておくと（デフォルトON）、**削除・移動・マージの実行前に自動でバックアップが作成**されます。

バックアップ先: `~/.obsidian-optimizer-backups/backup-[タイムスタンプ]/`

### 自動バックアップスケジュール

v4.3では、**毎日 or 毎週の自動バックアップスケジュール**を設定できるようになりました。「今すぐバックアップ」ボタンで手動実行も可能。定期的なバックアップを自動化することで、万が一の事態に備えられます。

### バックアップ管理

設定画面のバックアップ管理セクションから:
- 過去のバックアップ一覧を確認
- 任意のバックアップからVaultを復元
- 不要なバックアップを削除

### 変更プレビュー

最適化実行前に、削除予定ファイル・接続予定ノートの一覧を確認。個別にチェックを外して除外可能。

### 統一プレビューモーダル

全ての箇所でノートのプレビューが**統一されたモーダルウィンドウ**で表示されます。ノートの内容確認、AI機能の実行、お気に入り登録がすべてこのモーダルから操作可能です。

### トースト通知

操作結果（成功・エラー・警告）が**トースト通知**で即座にフィードバックされます。「ファイルが削除されました」「バックアップが完了しました」「APIキーが無効です」——操作の結果を見逃しません。

### Vaultの安全保護

- `isPathInsideVault()`関数により、**Vault外のファイルへの誤操作を防止**
- iCloud同期中（evicted）のファイルは500msのタイムアウトで安全に処理
- 二重起動防止（`requestSingleInstanceLock`）

## インストール方法（Mac版）

Mac版は `.zip` アーカイブで提供しています。以下の手順に沿ってインストールしてください。

### 手順1: zipファイルを展開する

### 手順2: 初回起動時のセキュリティ警告への対処

Obsidian Optimizerは現在Apple Developer Programによるコード署名を行っていないため、初回起動時にmacOSのセキュリティ機能（Gatekeeper）によって警告が表示される場合があります。

これはアプリが危険という意味ではなく、**Apple公式の審査を経ていない個人開発アプリに対して標準的に表示されるもの**です。

#### パターンA:「"Obsidian Optimizer"は、開発元を検証できないため開けません」と表示された場合

1. **システム設定**を開く（画面左上のリンゴマーク → システム設定）
2. 左メニューから**「プライバシーとセキュリティ」**を選択
3. 画面を下にスクロールすると、「"Obsidian Optimizer"は開発元を確認できないため、使用がブロックされました」というメッセージが表示されています
4. **「このまま開く」**ボタンをクリック
5. 確認ダイアログが表示されたら**「開く」**をクリック

**【🖼️ 画像㉓: macOSセキュリティ警告 → システム設定 → 「このまま開く」の手順スクリーンショット】**
*（3枚のスクリーンショットを横並びにして、矢印で手順を示す）*

#### パターンB:「"Obsidian Optimizer"が悪質なソフトウェアかどうかをAppleでは確認できないため、このソフトウェアは開けません」と表示された場合

上記パターンAと同じ手順で解決できます。

#### パターンC: 右クリックで開く方法（代替手段）

1. Finderでアプリケーションフォルダを開く
2. 「Obsidian Optimizer」を**右クリック**（またはControlキーを押しながらクリック）
3. メニューから**「開く」**を選択
4. 確認ダイアログで**「開く」**をクリック

この方法は一度だけ実行すれば、次回からは通常どおりダブルクリックで起動できます。

### 手順3: 起動を確認

アプリが起動すると、ダークテーマのダッシュボード画面が表示されます（ライトテーマへの切り替えも可能）。左側にサイドバー、中央にメインコンテンツが配置されたレイアウトです。

## トラブルシューティング（インストール関連）

### Mac: 「アプリケーションが破損しています」と表示される

ダウンロード時にmacOSが付与する「隔離属性」が原因です。ターミナルを開いて以下のコマンドを実行してください:

```
xattr -cr /Applications/Obsidian\ Optimizer.app
```

実行後、再度アプリを起動してください。

### Mac: アプリが開けない

macOSのセキュリティ設定により、アプリ自体がブロックされる場合があります。

1. アプリファイルを**右クリック** → **「開く」**を選択
2. 確認ダイアログで**「開く」**をクリック

### Windows: インストール後にアプリが起動しない

以下を確認してください:
1. Windows 10 64bit 以降であること（32bit版は非対応）
2. アンチウイルスソフトがブロックしていないか確認
3. 管理者権限で実行を試す（exeを右クリック → 「管理者として実行」）

### 共通: Vaultフォルダが認識されない

Obsidian OptimizerはVaultフォルダ内の `.obsidian` フォルダの存在で判定しています。以下を確認してください:
1. 指定したフォルダがObsidianのVaultフォルダであること
2. フォルダ内に `.obsidian` サブフォルダが存在すること
3. フォルダへのアクセス権限があること（特にiCloud Drive上の場合）

## よくある質問

**Q: Obsidian本体がなくても使えますか？**
A: Obsidian Optimizerは、Vaultフォルダ（.mdファイルの集まり）を直接操作します。Obsidian本体がインストールされていなくてもスキャン・分析は可能ですが、「Obsidianで開く」ボタンはObsidianが必要です。

**Q: ノートが消えてしまわないか心配です。**
A: デフォルトで自動バックアップが有効になっています。万が一の場合、設定画面のバックアップ管理から復元可能です。また、最適化実行前に変更プレビュー画面で影響範囲を確認できます。自動バックアップスケジュール（毎日/毎週）を設定しておくと、さらに安心です。

**Q: Obsidian Syncや他のクラウド同期と併用できますか？**
A: はい。Obsidian OptimizerはVaultフォルダ内のファイルを読み書きするだけなので、同期サービスとの競合はありません。ただし、同期中にファイルを操作すると競合が起きる可能性があるため、同期が完了してからスキャンすることをおすすめします。

**Q: iCloud Drive上のVaultでも動作しますか？**
A: はい。iCloud Driveの「ダウンロード待ち」状態のファイル（evicted file）にも対応しています。500msのタイムアウトを設けて安全に処理します。

**Q: Vaultのノートが非常に多い（5,000個以上）のですが、動作が遅くなりませんか？**
A: 数千ノート規模を想定して設計しています。スキャンはファイル単位で非同期処理し、UIのブロックを最小限に抑えています。10,000ノート以上の場合は、初回スキャンに数分かかる場合があります。

**Q: AI機能を使うにはどうすればいいですか？**
A: 「⚙️ 設定」→「🤖 AI設定」で、Claude / OpenAI / Gemini いずれかのAPIキーを入力してください。設定後、「🔗 接続テスト」で動作確認ができます。APIキーは端末内にのみ保存され、外部に送信されることはありません。

**Q: AI機能にはいくらかかりますか？**
A: AI機能の利用にはLLMプロバイダーのAPIキーが必要です。APIの利用料金はプロバイダー（Anthropic / OpenAI / Google）の料金体系に依存します。一般的な使い方（ノート要約、タグ提案など）であれば、月に数十円〜数百円程度です。設定タブの「AI利用状況」で累計コストを確認できます。

**Q: AI機能なしでも十分使えますか？**
A: もちろんです。AI機能はオプショナルな付加機能であり、スキャン・最適化・MOC生成・グラフビュー・整理ツール・タスク管理など、コア機能はすべてAI不要で利用できます。AI機能は「あればさらに便利」というポジションです。

**Q: タスクはどこに保存されますか？**
A: タスクはVault内の「📋 Tasks.md」ノートに自動保存されます。Obsidian上でも確認・編集できます。Obsidian Tasks プラグイン形式（`- [ ] タスク名 📅 YYYY-MM-DD`）で出力されるため、Tasksプラグインとも連携します。

**Q: Obsidianダッシュボード（ノート）が正しく表示されません。**
A: 生成されたダッシュボードノートはDataviewクエリを使用するため、ObsidianにDataviewプラグインがインストール・有効化されている必要があります。タスクボードの場合はTasksプラグインも必要です。「Obsidian → 設定 → コミュニティプラグイン → 検索 → インストール → 有効化」で導入できます。

**Q: 整理ツールの「一括スキャン」とは？**
A: 「🧹 整理ツール」タブの「🔍 全ツールを一括スキャン」ボタンを押すと、7つの整理ツール（メタデータ補完・Inbox振り分け・タイトルリネーム・リンク正規化・空フォルダ検出・TODO抽出・長文ノート分割）を一度にまとめて実行し、結果をサマリーカードで一覧表示します。

**Q: NotionやEvernoteからノートをインポートできますか？**
A: はい。「⚙️ 設定」→「📥 ノートインポーター」から、Notion（Markdownエクスポート）、Evernote（.enexファイル）、Bear（Markdownエクスポート）のインポートに対応しています。フォーマット変換とリンク修正が自動で行われます。

**Q: 実績バッジの条件は？**
A: 実績バッジはVaultの整理活動に応じて自動的にアンロックされるゲーミフィケーション要素です。「初回スキャン完了」「孤立ノート0達成」「連続7日スキャン」「MOC 10個作成」など、7つの実績があります。ダッシュボードの実績セクションで確認できます。

**Q: MOCテンプレートを他の人と共有できますか？**
A: はい。MOC作成タブの「テンプレート管理」から、カスタムテンプレートをJSON形式でエクスポートできます。他のユーザーは「📥 テンプレートを読み込み」ボタンでそのJSONファイルをインポートできます。

**Q: 今後のアップデートはどうなりますか？**
A: 購入者にはアップデート版を無料で提供します。アプリ内の「🔄 アップデート」→「更新を確認」ボタンで最新版の有無を確認できます。また、noteの記事を更新してダウンロードリンクを差し替えますので、定期的に記事を確認してください。

書くことに集中したいのに、整理に時間を取られる。MOCを作りたいのに、面倒で後回しにしてしまう。Vaultの全体像が見えず、同じトピックについて何度もノートを作ってしまう——。

ボタンひとつでスキャン。ワンクリックでMOC生成。AIでノートを深掘り。自動でリンク修復。リアルタイムで監視。タスクもまとめて管理。
